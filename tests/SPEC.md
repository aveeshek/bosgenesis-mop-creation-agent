# Tests Specification

## Intent

`tests/` defines future verification strategy.

## Required test groups

- Unit tests.
- MCP policy tests.
- Namespace blocking tests.
- Secret redaction tests.
- Prompt grounding tests.
- Document validation tests.
- Application-mode metadata-only tests.
- Observability/audit tests.
- Contract tests for generated artifacts.

## No-code constraint

No test code is included in the initial scaffold.
