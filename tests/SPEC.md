# Tests Specification

## Intent

`tests/` defines automated validation for the MoP Creation Agent.

## Required test groups

- Unit tests for configuration, orchestration, memory, retrieval, reconstruction,
  rendering, observability, and output contracts.
- API and MCP contract tests.
- Snapshot reader and selector tests.
- Resource classification and safety tests.
- Secret redaction and no-secret-artifact tests.
- Bounded LLM reasoning and repair suggestion tests.
- Qdrant reference lookup tests.
- PDF renderer tests.
- Observability/audit tests, including Langfuse v3 export compatibility and
  telemetry fail-open behavior.
- Contract tests for generated artifacts and machine execution plans.

## Validation reports

Phase 13.1 requires validation reports generated by:

```bash
./playbook/test-report.sh
```

or on Windows:

```powershell
powershell -ExecutionPolicy Bypass -File .\playbook\test-report.ps1
```

Generated outputs:

- `reports/pytest/test-results.xml`
- `reports/coverage/coverage.xml`
- `reports/coverage/html/index.html`

Generated reports are ignored by git.

## Coverage expectations

Coverage collection must target `bosgenesis_mop_creation_agent` with branch
coverage enabled. Current Phase 13.1 baseline is intended to keep broad module
visibility rather than enforce a hard minimum. Future phases may add a
`--cov-fail-under` threshold once unstable integration-heavy areas have more
deterministic tests.
