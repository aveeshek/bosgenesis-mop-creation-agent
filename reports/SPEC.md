# Reports Specification

## Intent

`reports/` is the local output directory for Phase 13.1 validation gates.

## Generated reports

The validation script writes:

- `reports/pytest/test-results.xml`
- `reports/coverage/coverage.xml`
- `reports/coverage/html/index.html`

Generated report artifacts are ignored by git. Only this specification and
`.gitkeep` are tracked.

## Required behavior

- Reports must be reproducible from `playbook/test-report.sh`.
- Test reports must be generated by pytest. The XML file may use the
  JUnit-compatible schema required by CI systems, but the path and documentation
  should call it a pytest report to avoid confusion with Java/JUnit.
- Coverage reports must include terminal, XML, and HTML output.
- Coverage collection must target `bosgenesis_mop_creation_agent`.
- Generated reports must not contain secrets, credentials, raw prompts, or
  production data.
